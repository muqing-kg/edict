## 🤔 关注了能看到什么？

### 🏛️ 架构拆解系列

公众号里主要写舰载系统的真实设计取舍：

- **为什么棱镜是关键质量闸门** —— 它拦下了多少不合格方案
- **云霄为什么负责分拣** —— Intent Detection 在实际系统里的落点
- **执行节点并行为什么不会互撞** —— 从权限矩阵到任务边界控制
- **SOUL.md 怎么写才稳定** —— 角色约束、流程约束和产出约束怎么组合

### 🔥 踩坑复盘

做 12 个 Agent 协作，每天都有新坑：

- ✍️ *Agent A 给 Agent B 发消息，B 直接拒收，因为权限矩阵没配对*
- 🔥 *Token 烧了 $50，结果棱镜把方案打回修订三次*
- 🐛 *中继路由任务后，执行节点集体不动，最后发现 agent ID 写错了*
- 💀 *install.sh 覆盖了用户自定义角色，后来补上了兼容与保留策略*

这些都是真实 Issue，文章里逐条复盘怎么修的。

### 💡 Token 省钱术

大家最关心的问题：**这套系统到底怎么控成本？**

公众号里会分享：
- 用 Claude 内置 cache 砍掉 80% SOUL.md 重复读取
- 「快速通道」机制：简单任务跳过棱镜校核
- 按需启动 Agent，不轮询不烧钱
- 精简 SOUL.md：1000 字以内搞定一个高质量角色
