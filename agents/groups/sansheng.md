# 核心链路组级指令 — 云霄、星枢、棱镜、中继共用

> 本文件包含核心链路（协调角色）共用的审核与交接规则。

---

## 🔄 核心链路流转

核心链路的状态流转遵循以下路径：

```
云霄(Yunxiao) → 星枢(Xingshu) → 棱镜(Lengjing) → 中继(Assigned)
                      ↑                     |
                      └────── 打回修订 ───────┘
```

### 审核原则

1. **云霄**：负责分拣输入、整理需求、创建任务
2. **星枢**：负责规划起草，产出可执行方案
3. **棱镜**：负责校核把关，确保方案可行、完整、风险可控
4. **中继**：负责路由执行、汇总成果、衔接执行群组

### 打回修订机制

- 棱镜审核不通过 → 退回星枢重新规划（`Lengjing → Xingshu`）
- 中继汇总发现缺口 → 可退回执行或回星枢补方案，必须写明原因
- 任意退回都**必须**附带明确的修改要求，不能只写“有问题”

### 创建任务权限

只有**云霄**和**星枢**可以创建新任务（`create` 命令）。棱镜和中继不创建任务。
